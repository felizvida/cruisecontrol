# Security Policy

## Scope

This repository is primarily a workflow, prompt, and configuration package for OpenCode and Codex-based research workflows. It does not ship a network service, but security issues can still matter, especially when they involve:

- secrets or tokens in configuration examples
- unsafe command suggestions
- accidental publication of local artifacts or sensitive paths
- misleading instructions that encourage insecure deployment or credential handling

## Reporting

Please avoid posting sensitive details in a public issue.

If you discover a security-sensitive problem, report it privately to the maintainer first. If GitHub private vulnerability reporting is enabled for the repository, prefer that channel.

## What Helps

When reporting, include:

- a short description of the issue
- the affected file or workflow
- impact
- a safe reproduction path, if one exists

## Disclosure

Once the issue is understood and fixed, coordinated public disclosure is welcome.
