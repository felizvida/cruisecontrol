# Data Package

This example ships the generated data needed to rebuild the paper's results tables and figures.

## Sample Data

- Local computed benchmark: [../results/demo_benchmark.json](../results/demo_benchmark.json)
- Episode-level trace export: [../results/episode_metrics.csv](../results/episode_metrics.csv)
- Machine-readable manifest: [dataset_manifest.json](dataset_manifest.json)

## What This Data Represents

The benchmark files are generated by the local simulator. They define:

- evaluation regimes
- per-method mission return
- distance traveled
- system cost of transport
- falls per 100 meters
- adaptation-compute energy
- adaptation calls
- cost-sweep and ablation slices

The paper is explicit that these values come from a lightweight stochastic deployment model rather than a real robot stack. The evidence is generated, but it should still be interpreted as simulation evidence rather than hardware evidence.
