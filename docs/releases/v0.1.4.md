# v0.1.4

Simulation-backed paper package and Biowulf workflow hardening release.

## Included

- Upgraded `examples/full-paper-sample/` from a hand-authored workflow demo into a reproducible stochastic simulation study
- Added executable benchmark generation in `results/simulate_battery_benchmark.py`
- Added episode-level output in `results/episode_metrics.csv`
- Rebuilt the sample paper around computed results and generated figures
- Replaced placeholder figure paths with real controller and tradeoff figure assets
- Added a new round snapshot at `paper/main_round4_simulation_package.pdf`
- Added the `biowulf-gpu` skill and command wrapper for one-node Biowulf GPU allocation
- Updated repo rules and templates so final papers require real executable computation
- Hardened Biowulf guidance around module loading, `lscratch`, and keeping heavy jobs off the frontend node

## Validation

- `examples/full-paper-sample/code/build_paper.sh` rebuilds the paper package from code
- `examples/full-paper-sample/code/validate_artifacts.sh` reports a clean 7-page PDF with no LaTeX warnings and embedded fonts
- `git diff --check` passes after the release cleanup

## Notes

- The full-paper sample is still a lightweight simulation study, not a hardware-validated robotics benchmark
- Early workflow artifacts inside `examples/full-paper-sample/` remain as provenance for the pre-simulation stages
- Biowulf guidance is now opinionated by default: allocate first, load modules inside the job, use scratch for large data, and never run heavy programs on the frontend
