# v0.1.6

paperreview.ai-backed paper review loop release.

## Included

- Added a new `paperreview-ai-review` skill for external paper review through `paperreview.ai`
- Added a bundled client script at `.opencode/skills/paperreview-ai-review/scripts/paperreview_client.py`
- Added an OpenCode wrapper command at `.opencode/commands/paperreview-ai-review.md`
- Updated `auto-paper-improvement-loop` to prefer `paperreview.ai` when configured and fall back to the local reviewer when needed
- Updated `paper-writing` so the full paper workflow knows about the new external review backend
- Updated repo and project metadata docs to encode the new review behavior
- Clarified that `paperreview.ai` currently needs an email at submission time, but later review retrieval is token-based once the token has been saved locally

## Validation

- `python3 -m py_compile .opencode/skills/paperreview-ai-review/scripts/paperreview_client.py`
- `python3 .opencode/skills/paperreview-ai-review/scripts/paperreview_client.py --help`
- `git diff --check`

## Notes

- This release wires in the external review backend and local artifact model, but does not include a live end-to-end upload trace against `paperreview.ai`
- The workflow now expects per-round saved review artifacts from the external backend when available:
  - `roundNN_paperreview_submission.json`
  - `roundNN_paperreview_response.json`
  - `roundNN_review.md`
  - `roundNN_scorecard.json`
- Two local scratch example folders remain intentionally untracked and were not included in the release:
  - `examples/opencode-zebrafish-brain-paper/`
  - `examples/opencode-zebrafish-brain-proteome-smoke/`
