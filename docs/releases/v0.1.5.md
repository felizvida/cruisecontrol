# v0.1.5

Codex-first routing and zebrafish JPR submission package release.

## Included

- Split the workflow surface into explicit pure-Codex and pure-OpenCode routes
- Added route-specific command wrappers for:
  - `research-pipeline-codex` and `research-pipeline-opencode`
  - `paper-writing-codex` and `paper-writing-opencode`
  - `paper-upgrade-codex` and `paper-upgrade-opencode`
- Made Codex the default route in the repo-level workflow docs and command guidance
- Tightened the review-driven paper loop so paper workflows persist explicit scored review artifacts as part of the finished package
- Added `WORKFLOW_ROUTES.md` as the route map for Codex versus OpenCode execution
- Added a new pure-Codex end-to-end example at `examples/codex-zebrafish-brain-paper-14rounds/`
- Preserved the full serialized paper loop for that example:
  - `paper/main_round0_original.pdf`
  - `paper/main_round1.pdf` through `paper/main_round14.pdf`
  - `review/round00_review.md` through `review/round13_review.md`
  - `review/ROUND_REVIEWS_LIVE.md`
- Tuned the zebrafish manuscript for `Journal of Proteome Research`
- Added submission support files:
  - `examples/codex-zebrafish-brain-paper-14rounds/submission/COVER_LETTER_JPR.md`
  - `examples/codex-zebrafish-brain-paper-14rounds/review/JPR_COMPARISON_2026-03-24.md`

## Validation

- `examples/codex-zebrafish-brain-paper-14rounds/code/build_paper.sh` rebuilt the final paper package
- `examples/codex-zebrafish-brain-paper-14rounds/code/validate_artifacts.sh` passed
- `pdfinfo examples/codex-zebrafish-brain-paper-14rounds/paper/main.pdf` reports the updated JPR-facing title metadata
- `grep -n 'Overfull\|Underfull\|undefined' examples/codex-zebrafish-brain-paper-14rounds/paper/main.log || true` returns no matches
- `git diff --check` passed before release

## Notes

- The Codex zebrafish paper is positioned honestly as a reproducible quantitative re-analysis, not a new wet-lab experiment
- The JPR benchmark comparison is based on public-source screening of recent `Journal of Proteome Research` papers and should be read as a submission-positioning aid, not a substitute for journal peer review
- The pure OpenCode route remains available, but Codex is now the default end-to-end path for this repo
