## Remote Server

- GPU: `4x A100 80GB`
- Modules: `module purge && module load cuda python`
- Conda: `eval "$(/opt/conda/bin/conda shell.bash hook)" && conda activate research`
- Code dir: `/home/you/research-project`
- Persistent data dir: `/data/you/research-project`
- Scratch dir: `/scratch/you` or job-local scratch
- Launch style: `screen`
- Notes: any quota, queue, or filesystem constraints

### Biowulf Example

- SSH: `ssh biowulf.nih.gov`
- Allocation style: `bash -lc 'srun --partition=gpu --gres=gpu:p100:2,lscratch:50 --cpus-per-task=14 --mem=32g --time=01:00:00 -N1 -n1 ...'`
- Modules: `module purge && module spider python && module spider pytorch && module load python/<version> && module load pytorch/<version>`
- Scratch dir: `/lscratch/$SLURM_JOB_ID`
- Persistent data dir: `/data/$USER/...` or project storage
- Limits: keep requests moderate and do not exceed one node
- Queue note: day-time waits can approach 2 hours; late-night waits are often shorter
- Verification: `hostname && nvidia-smi --query-gpu=index,name,memory.total --format=csv,noheader`
- Notes: run remote commands through `bash -lc` so Slurm tools are on `PATH`; never run heavy programs on the frontend; keep large data, caches, checkpoints, and temp outputs off `$HOME`

- Prefer local compute for quick pilots, smoke tests, and lightweight simulations.

- Record the actual launch path used for paper-critical runs: environment setup, modules, allocation command, code directory, and result artifact locations.
