# Support

## Best Place To Start

If you are trying to use the repo successfully on your machine, start with:

- `README.md`
- `QUICKSTART.md`
- `AUTO_RESEARCH_GUIDE.md`

## Good Issue Types

Open an issue if you hit:

- broken setup instructions
- MCP configuration mismatches
- unclear workflow behavior
- incorrect repo metadata or documentation

## What To Include

Support requests are much easier to resolve when they include:

- your command or workflow step
- the exact error message
- whether the issue is in OpenCode, Codex, or repo documentation
- whether the problem is machine-specific

## Non-Issue Cases

Please do not commit generated local artifacts unless they are intentionally documented examples.

For one-off usage questions, a discussion or issue with clear reproduction steps is usually enough.
